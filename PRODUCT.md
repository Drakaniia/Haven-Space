# Haven Space - Product Context

## Register

**product** - This is an application UI where design serves the product. Users come to complete tasks efficiently.

## Users

- **Boarders**: Students or young professionals looking for affordable accommodation
- **Landlords**: Property owners managing their listings and tenants
- **Administrators**: Platform managers overseeing the entire system

## Product Purpose

Haven Space is a property management platform that connects boarders with landlords, streamlining the rental process from listing to payment management.

## Brand Tone

- **Professional yet approachable**
- **Efficient and task-focused**
- **Trustworthy and reliable**
- **Clean and organized**

## Strategic Principles

1. **Task efficiency**: Users should complete their tasks with minimal friction
2. **Clear hierarchy**: Information should be organized by importance and frequency of use
3. **Consistent patterns**: Familiar UI patterns reduce cognitive load
4. **Responsive design**: Works across devices from mobile to desktop
5. **Accessibility**: WCAG AA compliance for all users

## Anti-References

- Avoid overly decorative elements that distract from tasks
- No complex animations that slow down workflows
- Avoid inconsistent component behaviors across different dashboards
- No dense walls of text - information should be scannable

## Key User Journeys

1. **Boarder**: Find room → Apply → Pay rent → Request maintenance
2. **Landlord**: List property → Review applications → Manage payments → Track maintenance
3. **Admin**: Monitor platform → Resolve disputes → Generate reports

## Success Metrics

- Time to complete common tasks
- User error rates
- Task completion rates
- User satisfaction with interface efficiency
